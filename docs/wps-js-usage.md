# WPS JS 报废差异查询使用说明

## 使用前准备

工作簿中需要保留两张原始数据表：

- `查询OA-存货报废申请单`
- `查询ERP-报废明细表`

预验证和查询都会在各自工作表的 `UsedRange` 前 20 行内自动识别表头，不再要求 OA、ERP 表头固定在某一行。

### 必需表头

OA 表必需表头：

- `表单编号`
- `金蝶云单据编号`
- `申请日期`
- `公司简称`
- `一级部门`
- `二级部门`
- `物料代码`
- `物料名称`
- `数量`
- `实际预算金额mx`

ERP 表必需表头：

- `单据编号`
- `日期`
- `源单单号`
- `区分公司简称`
- `一级部门`
- `二级部门`
- `物料编码`
- `物料名称`
- `实发数量`
- `总成本`

表头识别时只会去掉字段名前后的空格，字段正文必须与上面的模板完全一致。

完整设计可参考 `docs/superpowers/specs/2026-05-17-scrap-variance-wps-js-design.md`。

## 作为 WPS 加载项运行

当前项目只支持 WPS 加载项模式，不再维护旧的“把整段 JS 宏复制到 WPS JS 宏编辑器”使用路径。

首次准备：

```bash
npm install
```

类型检查：

```bash
npm run typecheck
```

构建加载项入口：

```bash
npm run build
```

启动调试：

```bash
npm run dev
```

`main.js` 由 esbuild 从 `src/entry.ts` 生成，但会提交到 git，方便 WPS 直接加载仓库根目录入口。每次修改 `src/**/*.ts` 后，都需要重新执行 `npm run build` 再回到 WPS 验证。

在 WPS 中加载成功后，功能区会出现 `报废差异工具`，包含操作按钮。查询条件不再放在功能区输入框里，点击 `查询当前页` 后会打开查询弹窗。

操作按钮：

- `预验证数据`：读取两张原始表，输出或刷新 `预验证结果`。
- `初始化输出表`：创建或刷新 `报废差异明细`、`OA视角单据对比`、`ERP视角单据对比` 三张输出表。
- `查询当前页`：只允许在 `报废差异明细`、`OA视角单据对比`、`ERP视角单据对比` 中使用，并且只刷新当前激活的输出表，其他输出表不联动。
- `展开物料`：在两张单据对比表中展开或收起当前汇总单据下方的物料行。
- `性能诊断`：读取当前工作簿和功能区筛选条件，把阶段耗时写入 `性能诊断结果`，不会替代正式查询结果。

验证顺序建议：

1. 打开包含 `查询OA-存货报废申请单` 和 `查询ERP-报废明细表` 的工作簿。
2. 点击 `预验证数据`，确认出现 `预验证结果`。
3. 点击 `初始化输出表`，确认出现 `报废差异明细`、`OA视角单据对比`、`ERP视角单据对比`。
4. 切换到要刷新的输出表，点击 `查询当前页`。
5. 在弹窗中填写本次查询条件；空白字段表示 all。
6. 点击 `查询`，确认只刷新当前激活的输出表。
7. 在 `OA视角单据对比` 或 `ERP视角单据对比` 选中 `行类型=汇总` 的单据行，点击 `展开物料` 查看物料级数量和金额；再次点击同一汇总行会收起。

## 查询

点击 `查询当前页` 只会在 `报废差异明细`、`OA视角单据对比`、`ERP视角单据对比` 中打开 `报废差异查询条件` 弹窗；当前表不是这三张输出表时，会在弹窗打开前拦截。弹窗包含：

- `公司简称`：可手输，也可从当前 OA/ERP 原始表读取到的候选中选择
- `一级部门`：可手输，也可从候选中选择
- `二级部门`：可手输，也可从候选中选择
- `开始日期`
- `结束日期`
- `查询方向`，只在 `报废差异明细` 可选；`OA视角单据对比` 和 `ERP视角单据对比` 中会禁用

弹窗打开时会优先带入当前输出表上次查询保存的条件；如果当前输出表还没有查询记录，则默认空白。空白表示 `all`，不限制该字段。候选下拉只是填写辅助，不限制输入；如果候选读取失败，仍可手输条件并查询。只填写 `公司简称=数控` 时，一级部门、二级部门和日期都不限制，只按公司筛选数控。

按钮行为：

- `查询`：按弹窗当前条件刷新当前激活输出表。
- `清空`：清空弹窗里的公司、部门、日期，并把查询方向恢复为 `OA金蝶单号查ERP`；不会恢复上次条件，适合快速改成查全部。
- `取消`：关闭弹窗，不刷新输出表。

点击 `初始化输出表` 会创建或刷新三张输出表：

- `报废差异明细`
- `OA视角单据对比`
- `ERP视角单据对比`

点击 `查询当前页` 只刷新当前激活的输出表，其他输出表不联动。

每张输出表的弹窗初始条件彼此独立。切换到另一张输出表再点 `查询当前页` 时，弹窗带入的是那张输出表自己的上次查询条件，不会带入前一张输出表的条件。

`报废差异明细` 读取 `查询方向`，支持：

- `OA金蝶单号查ERP`：按 OA 的申请日期、公司、一级部门、二级部门筛选，再用 `OA.金蝶云单据编号 = ERP.单据编号` 查 ERP。
- `ERP源单查OA`：按 ERP 的日期、区分公司简称、一级部门、二级部门筛选，再用 `ERP.源单单号 = OA.表单编号` 查 OA。

`OA视角单据对比` 和 `ERP视角单据对比` 不读取 `查询方向`，弹窗中该项不可填写，因为表名已经固定了起点视角。

`OA视角单据对比` 固定按 OA 表筛选和聚合：

- 日期条件解释为 `OA.申请日期`
- 公司条件解释为 `OA.公司简称`
- ERP 数量和金额按 `ERP.单据编号 = OA.金蝶云单据编号` 汇总

`ERP视角单据对比` 固定按 ERP 表筛选和聚合：

- 日期条件解释为 `ERP.日期`
- 公司条件解释为 `ERP.区分公司简称`
- OA 数量和金额按 `OA.表单编号 = ERP.源单单号` 汇总

在 `OA视角单据对比` 或 `ERP视角单据对比` 中选中 `行类型=汇总` 的单据行，再点击 `展开物料`，会在该行下面插入该单据的物料行；再次点击同一汇总行会收起物料行。物料行的数量、金额和差额都是该物料自己的聚合值，不复用汇总行数值。

每张输出表会在隐藏元数据中记录本表上次查询使用的筛选条件。切换到另一张输出表重新查询后，回到原输出表展开物料时，仍按原输出表自己的查询条件展开，不受其他输出页或下一次弹窗输入影响。

## 性能与输出约束

- OA、ERP 工作表通过 `UsedRange.Value2` 一次性批量读取。
- 查询和预验证结果通过二维数组批量写回。
- 如果整块批量写入失败，会自动回退到分块批量写入。
- 大结果表禁止按行逐条或按单元格逐格写回。
- 查询结果不再固定清理到 `200000` 行。工具会在隐藏元数据中记录每张输出表上次写入范围，下次只清理当前表上次由工具生成的区域。
- 预验证输出固定清理 `A1:H200000`。

## 性能基准与诊断

开发环境可以运行 Node benchmark：

```bash
npm run bench
npm run bench -- --scale stress
```

默认 benchmark 跑 `1万 / 5万` 行模拟数据；`--scale stress` 会追加 `20万` 行压力档。报告会写入 `bench-results/latest.json`，该文件是本机结果，不作为稳定产物提交。

`.bench/` 和 `bench-results/latest.json` 是本地生成产物，已配置为忽略文件。

WPS 功能区中的 `性能诊断` 按钮会读取当前工作簿的 OA、ERP，并把阶段耗时写入 `性能诊断结果`。诊断不会清空或覆盖三张正式输出表的查询结果。

诊断表中的 `不适用` 表示该行不是阶段统计，例如运行时能力探针或结果规模摘要；`无确切信息` 表示该指标理论上有意义，但当前 WPS 运行时没有提供可靠数据。

`内存MB` 表示阶段前后 JS heap 使用量差值。若运行时提供 `performance.memory.usedJSHeapSize`，诊断会写入该口径的差值；若 WPS 环境无法提供可靠内存数据，诊断表会显示 `无确切信息`。

## 常见提示与错误

- `找不到工作表`：检查原始表名是否被改动。
- `缺少关键列`：检查表头文字是否和上面的必需表头一致。
- `日期格式不正确`：使用 `2026-05-01` 或 `2026/5/1` 格式。
- `查询方向不正确`：弹窗 `查询方向` 只能使用 `OA金蝶单号查ERP` 或 `ERP源单查OA`，也可以留空使用默认方向。
- `金蝶云单据编号为空`：预验证提醒，不阻断查询；在 `OA金蝶单号查ERP` 中会表现为 `OA有申请，ERP无出库`。
- `查询条件没有匹配到 OA 数据`：`OA金蝶单号查ERP` 没有匹配到 OA 数据，放宽公司、部门或日期条件后再查。
- `查询条件没有匹配到 ERP 数据`：`ERP源单查OA` 没有匹配到 ERP 数据，放宽公司、部门或日期条件后再查。
- `当前工作表不支持查询或展开`：切换到 `报废差异明细`、`OA视角单据对比` 或 `ERP视角单据对比` 后再点击 `查询当前页`；展开物料仍只支持两张单据对比表。
- `当前工作表不支持展开物料`：只有 `OA视角单据对比` 和 `ERP视角单据对比` 支持展开物料。
- `当前输出表缺少查询条件记录`：当前页没有可用于展开物料的隐藏查询条件记录，先在当前页点击 `查询当前页`，再选中汇总行展开。
