# WPS JS 报废差异查询使用说明

## 使用前准备

工作簿中需要保留两张原始数据表：

- `查询OA-存货报废申请单`
- `查询ERP-报废明细表`

OA 表头在第 3 行，ERP 表头在第 1 行。

### 必需表头

OA 表必需表头：

- `表单编号`
- `申请日期`
- `公司简称`
- `一级部门`
- `二级部门`
- `物料代码`
- `物料名称`
- `数量`
- `实际预算金额mx`

ERP 表必需表头：

- `单据编号`
- `日期`
- `源单单号`
- `区分公司简称`
- `一级部门`
- `二级部门`
- `物料编码`
- `物料名称`
- `实发数量`
- `总成本`

完整设计可参考 `docs/superpowers/specs/2026-05-17-scrap-variance-wps-js-design.md`。

## 作为 WPS 加载项运行

当前项目已经整理为轻量 WPS 加载项。加载项只提供功能区按钮入口，核心仍然是在当前工作簿中读取 OA/ERP 两张表并写回结果。

首次准备：

```bash
npm install
```

启动调试：

```bash
npm run dev
```

在 WPS 中加载成功后，功能区会出现 `报废差异工具`，包含三个按钮：

- `预验证数据`: 读取两张原始表，输出或刷新 `预验证结果`。
- `初始化查询面板`: 创建或刷新 `查询面板`，用于填写筛选条件。
- `执行差异查询`: 读取 `查询面板` 的 B2:B6 条件，输出汇总差异和明细差异。

验证顺序建议：

1. 打开包含 `查询OA-存货报废申请单` 和 `查询ERP-报废明细表` 的工作簿。
2. 点击 `预验证数据`，确认出现 `预验证结果`。
3. 点击 `初始化查询面板`，确认出现 `查询面板`。
4. 在 `查询面板` 的 B2:B6 填写筛选条件。
5. 点击 `执行差异查询`，确认 `查询面板` 中出现 `汇总差异` 和 `明细差异`。

## 安装宏（旧方式）

1. 打开 `存货报废流程-erp&oa对比表.xlsx`。
2. 打开 WPS 的 JS 宏编辑器。
3. 将 `src/macros/scrap-variance-query.js` 的完整内容粘贴进去。
4. 运行 `setupQueryPanel()`，创建 `查询面板`。

## 查询

在 `查询面板` 填写：

- `B2`: 公司简称，可以为空。
- `B3`: 一级部门，可以为空。
- `B4`: 二级部门，可以为空。
- `B5`: 开始日期，对应 OA 表 `申请日期`，可以为空。
- `B6`: 结束日期，对应 OA 表 `申请日期`，可以为空。

查询以 OA 申请为主线：

- 已匹配记录通过 `ERP.源单单号 = OA.表单编号` 关联。
- 已匹配记录只按 OA 条件筛选，不按 ERP `日期` 二次过滤，避免跨期出库被误判。
- 只有 `ERP出库对应OA未在当前OA数据中找到` 这类 ERP-only 记录，才按 ERP 表 `日期` 参与筛选。

运行：

```js
runScrapVarianceQuery()
```

结果会写入 `查询面板`：

- `汇总差异`: 按公司和部门汇总。
- `明细差异`: 按 OA 表单编号和物料编码追到明细。

## 按钮绑定

可以在 `查询面板` 插入一个按钮或形状，并把宏绑定到：

```js
runScrapVarianceQuery
```

如果 WPS 版本不支持直接绑定 JS 函数，就在 JS 宏编辑器中手动运行 `runScrapVarianceQuery()`。

## 常见提示与错误

- `找不到工作表`: 检查原始表名是否被改动。
- `缺少关键列`: 检查表头文字是否和上面的必需表头一致。
- `日期格式不正确`: 使用 `2026-05-01` 或 `2026/5/1` 格式。
- `查询条件没有匹配到 OA 数据`: 放宽公司、部门或日期条件后再查。
