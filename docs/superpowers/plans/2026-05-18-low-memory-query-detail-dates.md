# Low Memory Query Detail Dates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add OA and ERP dates to the query panel detail output while keeping the first implementation memory-light and fast.

**Architecture:** Keep the WPS side unchanged: `UsedRange.Value2` bulk reads, fixed-range clear, and bulk matrix writes. Keep all date work inside the existing pure TypeScript aggregation and comparison passes. Store date and ERP document aggregates as joined strings in each grouped row so the query does not allocate per-group date arrays, per-group date sets, or a compare-time union key array.

**Tech Stack:** TypeScript, Vitest, esbuild, decimal.js-light, WPS JS add-in API.

---

## Scope Check

The approved behavior change is limited to the query panel detail section:

- Add `OA申请日期` and `ERP日期` to detail rows.
- Keep summary rows unchanged.
- Keep filtering, difference classification, precheck rules, sheet names, and WPS add-in entrypoints unchanged.
- Keep generated root `main.js` committed after source changes.

This plan also includes internal memory reductions that preserve output behavior:

- Replace `ErpAggRow.erpDocNumbers: string[]` with a joined string.
- Avoid building a `unionKeys()` array and `seen` set during comparison.
- Reuse the already-normalized `dateKey` inside each row pass.

```ts
export function appendUniqueJoinedText(currentText: string, nextText: string, delimiter = "、"): string {
  const current = normalizeText(currentText);
  const next = normalizeText(nextText);

  if (!next) {
    return current;
  }
  if (!current) {
    return next;
  }
  if (
    current === next ||
    current.startsWith(`${next}${delimiter}`) ||
    current.endsWith(`${delimiter}${next}`) ||
    current.includes(`${delimiter}${next}${delimiter}`)
  ) {
    return current;
  }

  return `${current}${delimiter}${next}`;
}
```

function addErpRowToGroup(
  result: Map<string, ErpAggRow>,
  row: RawRow,
  sourceFormNumber: string,
  itemCode: string,
  dateKey: string
): void {
  const key = makeDetailKey(sourceFormNumber, itemCode);

  if (!target) {
    target = {
      sourceFormNumber,

      itemName: normalizeText(row["物料名称"]),
      company: normalizeText(row["区分公司简称"]),
