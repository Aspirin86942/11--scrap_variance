# WPS JS 报废差异查询使用说明

## 使用前准备

工作簿中需要保留两张原始数据表：

- `查询OA-存货报废申请单`
- `查询ERP-报废明细表`

预验证和查询都会在各自工作表的 `UsedRange` 前 20 行内自动识别表头，不再要求 OA、ERP 表头固定在某一行。

### 必需表头

OA 表必需表头：

- `表单编号`
- `申请日期`
- `公司简称`
- `一级部门`
- `二级部门`
- `物料代码`
- `物料名称`
- `数量`
- `实际预算金额mx`

ERP 表必需表头：

- `单据编号`
- `日期`
- `源单单号`
- `区分公司简称`
- `一级部门`
- `二级部门`
- `物料编码`
- `物料名称`
- `实发数量`
- `总成本`

表头识别时只会去掉字段名前后的空格，字段正文必须与上面的模板完全一致。

完整设计可参考 `docs/superpowers/specs/2026-05-17-scrap-variance-wps-js-design.md`。

## 作为 WPS 加载项运行

当前项目只支持 WPS 加载项模式，不再维护旧的“把整段 JS 宏复制到 WPS JS 宏编辑器”使用路径。

首次准备：

```bash
npm install
```

类型检查：

```bash
npm run typecheck
```

构建加载项入口：

```bash
npm run build
```

启动调试：

```bash
npm run dev
```

`main.js` 由 esbuild 从 `src/main.ts` 生成，但会提交到 git，方便 WPS 直接加载仓库根目录入口。每次修改 `src/**/*.ts` 后，都需要重新执行 `npm run build` 再回到 WPS 验证。

在 WPS 中加载成功后，功能区会出现 `报废差异工具`，包含三个按钮：

- `预验证数据`：读取两张原始表，输出或刷新 `预验证结果`。
- `初始化查询面板`：创建或刷新 `查询面板`，用于填写筛选条件。
- `执行差异查询`：读取 `查询面板` 的 B2:B6 条件，输出汇总差异和明细差异。

验证顺序建议：

1. 打开包含 `查询OA-存货报废申请单` 和 `查询ERP-报废明细表` 的工作簿。
2. 点击 `预验证数据`，确认出现 `预验证结果`。
3. 点击 `初始化查询面板`，确认出现 `查询面板`。
4. 在 `查询面板` 的 B2:B6 填写筛选条件。
5. 点击 `执行差异查询`，确认 `查询面板` 中出现 `汇总差异` 和 `明细差异`。

## 查询

在 `查询面板` 填写：

- `B2`：公司简称，可以为空。
- `B3`：一级部门，可以为空。
- `B4`：二级部门，可以为空。
- `B5`：开始日期，对应 OA 表 `申请日期`，可以为空。
- `B6`：结束日期，对应 OA 表 `申请日期`，可以为空。

查询以 OA 申请为主线：

- 已匹配记录通过 `ERP.源单单号 = OA.表单编号` 关联。
- 已匹配记录只按 OA 条件筛选，不按 ERP `日期` 二次过滤，避免跨期出库被误判。
- 只有 `ERP出库对应OA未在当前OA数据中找到` 这类 ERP-only 记录，才按 ERP 表 `日期` 参与筛选。

## 性能与输出约束

- OA、ERP 工作表通过 `UsedRange.Value2` 一次性批量读取。
- 查询和预验证结果通过二维数组批量写回。
- 如果整块批量写入失败，会自动回退到分块批量写入。
- 大结果表禁止按行逐条或按单元格逐格写回。
- 查询输出固定清理 `A8:O200000`。
- 预验证输出固定清理 `A1:H200000`。

## 常见提示与错误

- `找不到工作表`：检查原始表名是否被改动。
- `缺少关键列`：检查表头文字是否和上面的必需表头一致。
- `日期格式不正确`：使用 `2026-05-01` 或 `2026/5/1` 格式。
- `查询条件没有匹配到 OA 数据`：放宽公司、部门或日期条件后再查。
